### CHAPTER 2 -- TEXT

● Structural markup: the elements that you can use to
describe both headings and paragraphs
● Semantic markup: which provides extra information; such
as where emphasis is placed in a sentence, that something
you have written is a quotation (and who said it), the
meaning of acronyms, and so on.

STRUCTURAL MARKUP

Headings go from h1 to h6 in order of size, decreasing.
<b> bold </b>
<i> italic <i>
<sup> superscript, goes above text, like power levels.</sup>
<sub> subscript, goes below text, like physics formulas.</sub>
<br /> --> '\n' NEWLINE
<hr /> --> TABLINE

SEMANTIC MARKUP

<strong>Text inside here is important. It appears bold.</strong>
<em>Em for emphasis is printed in italic, it sublely changes the meaning of a sentence (supposedly).</em>
<blockquote cite="http://wiki.whatever.com/something-something"><p>Notice how the paragraph is inside the quote. 
This is for larger quotes.</p></blockquote>
<q>This is for smaller quotes inside a paragraph.</q>
<p><abbr title="Professor">Prof.</abbr> Oak asked Ash to pick one of the three pokemon balls.</p> --> Acronyms and abbreviations
<p><cite>Blue Pale Dot</cite> by Carl Sagan is a book about smoking weed and looking at the stars.</p>
<p> To know what a <dfn> Black Hole </dfn> is you should ask your mom. </p> --> Defining concepts.
<address><p><a href="mailto:andralv77@gmail.com"> ETC ETC </p></address> --> Displays information about ME.
<p>This was the <del> worst </del> <ins> best </ins> idea she ever had. del for 'delete', ins for 'insert' (pretty useless)
<s>Something that is no longer accurate or relevant goes here. Like my life.</s>

### CHAPTER 6 -- TABLES

<table> everything table-like goes here. This is the structure, it is constructed row by row. </table>
	<tr>
		<td>23</td>
		<td>11</td>
		<td>32</td>
	</tr>
</table>
<th> -- table heading, it's the title for each column or row. <th scope="row"> <th scope="col"> -- Appears in bold
column and row spanning -- it's like merging cells, so they take up more than one space.
<td colspan="2"> whatever </td> -- the following elements should be less to make up for that space ---*
<td rowspan="2"> whatever </td> -- the following elements should be less to make up for that space ---*

long tables --> separate top from middle and bottom
<thead>heading goes here</thead>
<tbody>everything else goes here</tbody>
<tfoot>a total or final line goes here</tfoot>
There used to be table attributes for width, columnspanning and columnspacing but there are frowned upon in html5 and 
should be left to CSS.
Same with border and bgcolor='hex code'

### CHAPTER 8 -- EXTRA MARKUP

doctypes: first thing in a website, specifies the version of html or xhtml being used.

Doctype for html5 --> <!DOCTYPE html>
comments <!-- whatever --> separate sections
id = "whatever" used on other tags, use it to style stuff in css and js
class = just like id but used to group and separate elements-pieces of code
<h#> <p> <li> and <ul> amongst others are block elements, they start in a new line, just so you know
<a> <b> <img> and <em> amongst others are inline elements, their instantiation doesnt make a new line
<div> to separate all parts of code, header, body, etc, end with comment
<span> is used inline to separate text from the rest, to later apply css, use it with id or class
<iframe> -- it is an applet or window to another website, commonly used for google maps
	src specifies the web address 
	height and width are dimensions
	scrolling adds scrollbars but dont use it since its not html5 -- no and yes
	frameborder -- adds a frame, not html5 -- 0 and 1
	seamless -- use it for no reason, it is html5
<meta> -- inside the head element, describes info about the page	
	defined values
		description -- explains what the website does, appears on search engines
		keywords -- these are like tags
		robots --noindex if i want to keep it private for google, nofollow if i want to keep
		it private for google links
	http-equiv
		author -- defines author of the site
		pragma -- avoids the browser storing cache
		expires -- use it to expire the cache

<!DOCTYPE html>
<html>
	 <head>
		 <title>Information About Your Pages</title>
		 <meta name="description"
			 content="An Essay on Installation Art" />
		 <meta name="keywords"
			 content="installation, art, opinion" />
		 <meta name="robots"
			 content="nofollow" />
		 <meta http-equiv="author"
			 content="Jon Duckett" />
		 <meta http-equiv="pragma"
			 content="no-cache" />

	 </head>
	 <body>
	 </body>
</html>

Using HTML5 for videos <video> tag:
	Always convert video to webm

<!DOCTYPE html>
<html>
	 <head>
		 <title>Adding HTML5 Video</title>
	 </head>
	 <body>
		 <video src="video/puppy.mp4"
			 poster="images/puppy.jpg"
			 width="400" height="300"
			 preload
			 controls
			 loop>
			 <p>A video of a puppy playing in the snow</p>
		 </video>
	 </body>
</html>

** src --> path
** poster --> thumbnail that shows while it is loading
** width and height are duh
** preload --> none, auto, or metadata, first waits for playback, second starts the video right away, third the browsers just collects info
** controls --> let browser use its own methods for handling the video
** autoplay --> starts video when page loads
** loop --> loops the video

To add video in many formats: 
<!DOCTYPE html>
<html>
	 <head>
		 <title>Multiple Video Sources</title>
	 </head>
	 <body>
		 <video poster="images/puppy.jpg" width="400"
			 height="320" preload controls loop>
			 <source src="video/puppy.mp4" type='video/
				 mp4;codecs="avc1.42E01E, mp4a.40.2"' />
			 <source src="video/puppy.webm" type='video/
				 webm;codecs="vp8, vorbis"' />
			 <p>A video of a puppy playing in the snow</p>
		 </video>
	 </body>
</html>

### CHAPTER 17 - HTML5 LAYOUT

Usual workflow --> div elements take care of everything --> default blocks --> page, header, nav, content, sidebar, article, footer
HTML5 worklow --> elements that have these names --> header, article, aside, footer, nav, etc...

well, pretty much you should put corresponding things inside these tags, here are some goodies:

- header and footer, as such
- nav --> sometimes used in the bottom, policy contact etc
- article --> can nest other articles
- aside --> if inside an article, pull a quote or something related to the article, if not, links or search bar or nav bar
- section --> wrapper for related elements, for instance a section can hold many articles
- hgroup --> contains several h1-h6 elements, like a title and its description
- figure --> img, or diagram or video or whatever related to an article, strictly this
- div --> contains the whole page (wrapper) and everything else
- a --> other than links, to link to an specific part of the page, can hold blocks

To have older browsers understand this, a css/html-js script is imminent, use it:
